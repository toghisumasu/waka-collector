# プロンプト文言一覧（script/dryrun_hyakuin.rb）

其の三十三時点（commit `3cd3e90`）での、LLM（qwen3:8b）に渡している全プロンプト・フィードバック文言を整理したもの。

## 1. 通常モード：`build_prompt`（generate API）

`OllamaClient.generate` に単発プロンプトとして渡すテンプレート本体（150〜204行目）。

```
あなたは連歌師（れんがし）です。次の句（前句）に付く「付句（つけく）」を一句作れ。

前句：{maeku_word}
句番号：{verse_no}句目
句型：{mora_desc}の付句を作れ
bui（部立・複数可）は必ず次から選べ：{valid_bui_categories.join('・')}
{forbidden_season_block}{season_instruction}
{kigo_line}
{kinshi_line}
{feedback_line}
【出力形式】
以下のJSONのみを出力せよ。前後の説明文・コードブロック不要。
{"ku":"付句の本文","mora":音数,"season":"春か夏か秋か冬か雑","bui":["部立1","部立2"],"tsuki":false,"hana":false}

tsuki は月を詠み込んだとき true、hana は桜の花を詠み込んだとき true
【最重要】mora は{mora_desc}を厳守せよ。

JSON:
```

### 行ごとの差し込み内容

| プレースホルダ | 内容 | 常時/条件付き | コード位置 |
|---|---|---|---|
| `mora_desc` | 長句なら「17音（五七五）」、短句なら「14音（七七）」 | 常時 | L152 |
| `valid_bui_categories` | `ShikimokuChecker.rules.keys + [時分, 人倫]`（正規カテゴリのみ、下位語は不可） | 常時 | L191, L297（其の三十三 T2で下位語混入を排除） |
| `forbidden_season_block` | 「【絶対禁止】season フィールドに"◯◯"を入れてはならない。"季語例"等の語を含む句も禁止。違反した場合は採点外とする。」 | `forbidden_seasons`ありの時のみ | L175-181 |
| `season_instruction` | 転換指示／継続指示／現季提示／自由、の4パターン（下記2参照） | 常時（内容は分岐） | L117-126 |
| `kigo_line` | 「参考季語（春）：霞・梅・桜」等、前句に含まれず禁止語でもない季語を最大3件 | 該当季語がある時のみ | L163-172 |
| `kinshi_line` | 「禁止部立（これらの語は避けよ）: 雨・雪・露…」 | `forbidden_bui`ありの時のみ | L128-133 |
| `feedback_line` | 「前回「{ku}」は{issue}。{message}」 | 直前attemptがreject された時のみ | L183（内容は下記3参照） |

## 2. `season_instruction` の4分岐（`filter_pool`内）

| 条件 | 文言 |
|---|---|
| `must_switch`（季の転換必須） | 「【季の転換】必ず{禁止季}以外の季にすること。選択肢：{残りの季一覧}」 |
| `must_continue`（同季継続必須） | 「★必ず{現在季}の季を継続すること（残り{n}句以上は続けよ）」 |
| 現在季あり（自由継続/転換可） | 「現在の季：{現在季}（継続または転換、どちらでも可）」 |
| 現在季なし | 「季は自由（雑・春・夏・秋・冬から選べ）」 |

## 3. `feedback_line`（reject理由別のメッセージ、次attemptのbuild_promptに差し込む）

| reject理由 | 発生条件 | message文言 | コード位置 |
|---|---|---|---|
| JSON解析エラー | LLM出力がJSONとしてパース不能 | 「正しいJSON形式で出力せよ」 | L420 |
| 重複句（初回） | 既出wordと完全一致 | 「以下は出力禁止（既出につき再使用不可）: {既出語一覧}。全く別の語・情景で新たに詠め」 | L437-438 |
| 重複句固着（duplicate_streak≥2） | 同一系統の重複が連続 | 上記＋「季語にとらわれず、無季（雑）の句として、季節を離れた情景・心情・人事を詠んでください」 | L444-445 |
| 禁止季 | LLMがJSONの`season`に禁止季を誤記入 | 「{禁止季一覧}以外の季にすること」 | L458-459 |
| 字余り（mora_error_streak<2、またはstreak≥2未満条件） | モーラ超過 | 「{target_mora}音になるよう、もっと短くしてください」 | L494 |
| 字足らず（ENDING_BY_MORA提示） | モーラ不足、streak<2または字足らずでstreak<3 | 「{現mora}音で{不足数}音足りません。句末に「{候補}」などを足すと{target_mora}音になります。」（候補が無い場合は「{target_mora}音になるよう、もっと長くしてください」） | L500-506 |
| 字数固着（mora_error_streak≥2かつ字余り、またはstreak≥3） | モーラ不一致が連続 | 「前回までの候補は字数が合いませんでした。季語を使わない雑（ぞう）の句として、全く新しい言葉で{target_mora}音の句を詠んでください。これまでの候補：{過去候補}」 | L487-490 |

※字足らず方向のみ「固着メッセージ」への切替閾値が `streak≥3`、字余り方向は `streak≥2`（其の三十二 Step B-3改の非対称化）。

## 4. chatモード（`OllamaClient.chat`、mora_error_streak閾値到達時の5往復会話）

固着（同じ句を繰り返す）を打開するための多段階チャット。字余り／字足らずで内容が分岐する（L334-404）。

### 4-1. 字余り方向（`last_mora_count > target_mora`）

| # | role | content |
|---|---|---|
| 1 | user | 「あなたはいま、同じような句を繰り返しています。直前の候補「{直前候補}」は{last_mora_count}音で、目標の{target_mora}音に合っていません。行き詰まっていることを認識してください。」 |
| 2 | assistant | 「はい、行き詰まっています。同じような句を繰り返しており、字数が合っていません。局面を打開する必要があります。」 |
| 3 | user | 「連歌では、季語（春・夏・秋・冬の語）を使う句と、季語を使わない「雑（ぞう）」の句があります。雑の句は季節に縛られず自由に詠めます。局面打開には雑の句が有効なことがあります。理解できましたか？」 |
| 4 | assistant | 「はい、理解しました。雑の句とは季語を含まない句で、季節に縛られず詠むことができます。」 |
| 5 | user | 「では局面打開のため、雑の句として全く新しい言葉で{target_mora}音の付け句を詠んでください。前句：{前句}／これまでの候補（再使用禁止）：{過去候補}／JSON形式で出力してください：{"ku":...,"bui":[...],"season":"雑","mora":{target_mora}}／buiには句の内容に応じた部立（降物・植物・水辺・山類・居所・旅・恋など）を実際に入れてください。季語がない場合でも風景・情景の部立は入ります。」 |

### 4-2. 字足らず方向（`last_mora_count <= target_mora`）

| # | role | content |
|---|---|---|
| 1 | user | 「あなたはいま、同じような句を繰り返しています。直前の候補「{直前候補}」は{last_mora_count}音で、目標の{target_mora}音より{不足数}音足りません。行き詰まっていることを認識してください。」 |
| 2 | assistant | 「はい、行き詰まっています。{不足数}音足りず、同じような句を繰り返しています。局面を打開する必要があります。」 |
| 3 | user | 「連歌では句の末尾に文末表現を加えて音数を整えます。{不足数}音足りない場合、{ending_hint}理解できましたか？」 |
| 4 | assistant | 「はい、理解しました。句末に文末表現を加えて{target_mora}音に整えます。」 |
| 5 | user | 「では句末に文末表現を加えて、全く新しい言葉で{target_mora}音の付け句を詠んでください。前句：{前句}／これまでの候補（再使用禁止）：{過去候補}／JSON形式で出力してください：{"ku":...,"bui":[...],"season":"季節を入れてください","mora":{target_mora}}」 |

`ending_hint` は `ENDING_BY_MORA[不足数]` があれば「句末に「{候補}」などを足すと自然です。」、無ければ「句末に文末表現を加えて長くしてください。」

## 5. `ENDING_BY_MORA` 定数（不足モーラ数→文末候補）

| 不足モーラ数 | 候補 |
|---|---|
| 1 | や・か・ぬ・に |
| 2 | かな・けり・らむ・なり |
| 3 | ぞかし・ぬかな・にけり |
| 4 | にけらし・ぬるかな・たるかな |
