# 第1部　全体構造 ― 百韻という一つの世界

## 1-1. 季節の弧（アーク）

百韻全体を通した季節の流れを俯瞰すると、一つの大きな弧が見える。

```
前半（1〜50句）           後半（51〜100句）
─────────────────────  ─────────────────────
春・秋が主役            冬の比重が増す
冬は点在               春・秋が収束へ向かう
夏は完全にゼロ          最後は春が「還ってくる」
```

**季節別の出現分布：**

| 季 | 句数 | 主な出現位置 |
|:---|---:|:---|
| 春 | 25 | 全折に分散。発句・名残折で象徴的に使用 |
| 秋 | 22 | 前半〜三折裏で充実。三折裏（66〜69）と名残折表（89〜91）が二大ピーク |
| 冬 | 10 | 後半に比重。三折以降で存在感が増す |
| 夏 | 0 | **完全排除** |
| 雑 | 43 | 全体の接着剤。述懐・恋・釈教・神祇を含む |

**重要な発見：春は「還ってくる」**

発句（1句）の春霞で開いた世界が、名残折裏（96〜98句）で「枯れし林に春風／霞む山／長閑な煙」という春の再来で閉じられる。これは偶然ではなく、**百韻を円環として閉じる骨法設計**である。

```
1句   雪ながら山本かすむ夕べかな    （春の開始）
  ↓
  …春・秋・冬・雑が織りなす世界…
  ↓
96句  かれし林も春風ぞふく          （春の再来）
97句  山はけさいく霜夜にかかすむらん（霞の再来＝発句と呼応）
  ↓
100句 人におしなべ道ぞたゞしき      （世界の意味の確定）
```

## 1-2. 序破急の構造

連歌・能楽に共通する「序破急」の枠組みで百韻を見ると、各折の役割が明確になる。

| 段階 | 折 | 句番 | 役割 | 特徴 |
|:---|:---|:---|:---|:---|
| **序** | 初折表 | 1〜8 | 世界を静かに開く | 最短安定型の季運び。式目に忠実 |
| **序** | 初折裏 | 9〜22 | 変奏を始める | 季構造が変化に富む。述懐の核が出現 |
| **破** | 二折表 | 23〜36 | 展開を広げる | クッションなし転季。恋が初出 |
| **破** | 二折裏 | 37〜50 | 主題を深める | 雑6句の多様化。釈教・神祇の対置 |
| **破→転** | 三折表 | 51〜64 | 技巧の頂点 | 体用回避が集中。視点の垂直移動 |
| **転** | 三折裏 | 65〜78 | 情感の頂点 | 秋の総決算。恋4句の集中 |
| **急** | 名残折表 | 79〜92 | 華やぎと深まり | 花の場（最高潮）。述懐の極限 |
| **急** | 名残折裏 | 93〜100 | 静かな収束 | 新素材を出さず世界を畳む |

**序破急の体感：**

- **序（初折）** ― 宗祇が慎重に骨格を組む。式目に忠実で「安全運転」
- **破（二折）** ― 肖柏・宗長が乗ってくる。展開が大胆になる
- **転（三折）** ― 全員の興が乗り、式目の限界まで攻める「際どい付け」が集中
- **急（名残折）** ― 花の場で華やぎ、最後は余韻の中に着地

## 1-3. 人間の内面の層 ― 恋と述懐の設計

景物（自然描写）だけでなく、人間の内面を扱う「恋」「述懐」がどう配置されているか。

### 恋の波

恋は「出しては引く」波のように設計されている。

| 出現 | 句番 | 句数 | 性質 |
|:---|:---|:---:|:---|
| 第1波 | 35〜38 | 4句（折跨ぎ） | 後朝（きぬぎぬ）から面影へ。濃密な入り |
| 第2波 | 45〜46 | 2句 | 露の消え帰り。秋風に溶ける |
| 第3波 | 71〜74 | 4句 | 待恋→疑念→宿命→恨み。恋の物語化 |
| 第4波 | 82〜83 | 2句 | 手枕の夢→契り絶ゆ。過去への遡行 |

**設計の妙：** 恋を一か所に固めず、4回に分けて配置。各波で恋の「角度」を変える（待つ・疑う・恨む・諦める）。これは百韻全体の感情の起伏を事前に構想しなければ実現しない。

### 述懐の深まり

述懐（無常・懐旧・老い）は、進むにつれて深く・長くなる。

| 出現 | 句番 | 句数 | 深度 |
|:---|:---|:---:|:---|
| 第1 | 10〜12 | 3句 | 孤独・無常の予感 |
| 第2 | 33〜34 | 2句 | この世の道の尽き |
| 第3 | 39〜44 | 6句 | 古都の恨み・釈教へ展開 |
| 第4 | 84〜88 | 5句 | 老い・隠遁・命の執着（最深） |

**設計の妙：** 述懐は百韻の「重し」。後半に向かって深まり、名残折表（84〜88）で極限に達する。この深まりがあるからこそ、名残折裏の春の再来が「救い」として効いてくる。

## 1-4. 円環としての百韻 ― 発句と揚句の呼応

| | 句 | 本歌 | 出典 |
|:---|:---|:---|:---|
| 発句 | 雪ながら山本かすむ夕べかな | 見渡せば山もと霞む水無瀬川夕べは秋となに思ひけむ | 後鳥羽院（新古今・春上） |
| 揚句 | 人におしなべ道ぞたゞしき | 奥山のおどろが下も踏み分けて道ある代ぞと人に知らせむ | 後鳥羽院（新古今・雑中） |

**入口と出口を同じ作者（後鳥羽院）の本歌で締める。** これにより百韻全体が院への奉納として一つの円環をなす。発句の「霞」が名残折裏（97句）で「霞む」として還り、揚句で「道」が確定する。

**当麻曼荼羅との類比：** 一枚の図に極楽の全体を階層的に収めるように、百韻は一座一日の張行の中に一つの世界を織り上げる。部分は全体の秩序に従い、全体は部分の積み重ねで完成する。

# 第2部　付合技法の類型 ― 8折を通じたカタログ

全100句・92の転じを分析した結果、繰り返し現れる技法が抽出できた。
これらは実装時に「LLMに何をさせるか」の仕様になる。

## 2-1. 技法カタログ（出現頻度順）

### 技法①　感覚モダリティの転換【最頻出】

前句の感覚（視覚・聴覚・嗅覚・触覚）を判定し、次句で別の感覚に転じる。

| 実例 | 転換 |
|:---|:---|
| 1→2 | 視覚（夕景）→ 嗅覚（梅の香）・聴覚（水音） |
| 29→30 | 視覚（霧の里）→ 聴覚（衣打つ声） |
| 66→67 | 視覚（旅の月）→ 触覚（霜しをる袖） |

**実装：** C層。前句の感覚種別をLLMに渡し、異なる感覚での生成を促す。

### 技法②　視点の移動（遠近・高低・内外）【高頻度】

前句の空間的性質を判定し、次句で反対方向に転じる。

| 実例 | 転換 |
|:---|:---|
| 5→6 | 空（月）→ 地（霜の野原） |
| 59→63 | 地→水面→空→山（垂直の弧） |
| 99→100 | 内（身の修養）→ 外（人の道） |

**実装：** C層。前句の空間ヒント（遠/近/高/低/内/外）をLLMに渡す。

### 技法③　景と心情の往復【高頻度】

景物句（自然描写）と心情句（述懐・恋）を交互に転じる。

| 実例 | 転換 |
|:---|:---|
| 9→10 | 景（嵐の山里）→ 心情（慣れぬ住まいの寂しさ） |
| 12→13 | 心情（無常）→ 景（露と花） |

**実装：** C層。前句が体（景物）か用（心情）かを判定し、反転を促す。

### 技法④　多義語による橋渡し【重要技法】

一つの語が二つの意味を同時に担い、異なる世界を接続する。

| 実例 | 多義語 | 接続する世界 |
|:---|:---|:---|
| 34→35 | 待つ | 述懐（行き場なし）⇔ 恋（後朝の待機） |
| 44→45 | 消え帰り | 釈教（輪廻）⇔ 恋（はかない逢瀬） |
| 74→75 | 世 | 恋（恋の世）⇔ 雑（山賤の世界） |

**実装：** C層の最高度。LLMに「前句末の語の多義性を活かして転じよ」と指示。
4回出現した重要技法だが、Rubyでは生成不可能。LLM固有の領域。

### 技法⑤　写実的帰結【論理的連想】

前句の状況から物理的・論理的に導かれる結果を次句にする。

| 実例 | 連想 |
|:---|:---|
| 7→8 | 草が枯れる → 見通しが良くなり垣根越しに道が見える |
| 15→16 | 鳥が帰る → 人も深山を行く |

**実装：** C層。前句のキーワードから物理的帰結を生成させる。

### 技法⑥　大転換（無常から再生など）【構造的頂点】

対極にある主題を一気に転換する。百韻の山場に配置される。

| 実例 | 転換 |
|:---|:---|
| 95→96 | 釈教（仏の隠顕・無常）→ 春（枯林に春風・再生） |

**実装：** C層。文脈の蓄積が必要。百韻全体での位置（急に入ったか）を意識させる。

## 2-2. 技法の層別分布

92転じを役割分担の観点で集計した結果：

| 層 | 関与した転じ数 | 割合 |
|:---|---:|---:|
| A層単独で解決（式目のみ） | 0 | 0% |
| A層が関与（句去・句数の確認） | 全転じ | 100%（潜在的） |
| B層が関与（辞書照合） | 約12 | 13% |
| C層が主体（詩的判断） | 92 | 100% |

**結論：付合の「面白さ」は100%がC層にある。**
A層は「式目違反を防ぐガードレール」として常に潜在的に機能するが、付合そのものを生成する力は持たない。**LLMに任せるべきは技法①〜⑥であり、Ruby は技法を生成するのではなく、生成結果を検査する役割に徹する。**

# 第3部　実装設計への統合

## 3-1. 三層アーキテクチャの確定

分析を通じて、役割分担が明確になった。

```
┌─────────────────────────────────────────┐
│  A層：Ruby / MeCab（決定論）              │
│  ・モーラ数判定（KuValidator）実装済み✅  │
│  ・句去の間隔チェック                     │
│  ・句数（季の連続上限・最低数）           │
│  ・一座一句物の重複チェック               │
│  ・打越チェック                           │
│  → 役割：式目違反を防ぐガードレール       │
└─────────────────────────────────────────┘
              ↑ 検査結果を渡す
┌─────────────────────────────────────────┐
│  B層：Ruby + 辞書（半決定論）            │
│  ・季語の辞書照合                         │
│  ・部立の辞書照合（体用フラグ付き）       │
│  ・語幹重複フィルタ                       │
│  → 役割：候補の絞り込み・前処理           │
└─────────────────────────────────────────┘
              ↑ 制約条件を渡す
┌─────────────────────────────────────────┐
│  C層：ローカルLLM（創造的判断）          │
│  ・技法①〜⑥の付合生成                    │
│  → 役割：付合の「面白さ」を生み出す       │
└─────────────────────────────────────────┘
```

## 3-2. 実装で新たに判明した必須仕様

折別分析から、当初の設計になかった重要仕様が判明した。

### 仕様1　体用フラグの必須化【最重要】

三折表・三折裏で「際どい句去」が集中したが、すべて「体と用の分別」で回避されていた。

**例：** 56句「露」（植物の用）→ 57句「時雨」（山類の用）は降物が1句間隔だが、両者が降物の「体」ではなく他部立の「用」として機能するため句去に抵触しない。

**→ B層の部立辞書には、各語に「体／用」のフラグと「どの部立の用か」を登録する必要がある。**

```
例：辞書エントリの構造（イメージ）
  時雨:
    primary_bui: 降物    # 本来の部立
    season: 冬
    yo_of: [山類, 植物]  # 「〜の用」として機能しうる部立
```

これがなければ、Ruby は正しい付合を「違反」と誤判定する。

### 仕様2　花の特別扱い【A層に追加ルール】

名残折表80〜81句「桜→花」の連続は、式目の「花＝一座四句物」により合法。

**→ A層に「桜と花の連続は植物句去の例外」というルールを実装する。**
さらに「灯火は時分に嫌わず（夜分の道具であり時分そのものではない）」も同様の例外ルール。

### 仕様3　折を跨いだ文脈管理【C層の前提】

恋の波・述懐の深まり・季の負債返済（三折裏→名残折表の春持ち越し）など、
**複数折を跨いだ設計**が随所にある。

**→ セッション管理に「現在どの折・どの段階（序破急）か」「恋・述懐の直近出現位置」を保持し、LLMのプロンプトに注入する。**

### 仕様4　序破急に応じた生成方針【C層のヒント】

| 段階 | LLMへのヒント |
|:---|:---|
| 序（初折） | 式目に忠実に。最短安定型の季運び |
| 破（二折） | 展開を広げる。転季を大胆に |
| 転（三折） | 技巧を凝らす。多義語・視点移動を積極使用 |
| 急（名残折） | 花の場で華やぎ、終盤は新素材を抑え余韻を出す |

## 3-3. 実装優先順位（改訂版）

折別分析の知見を反映した優先順位。

```
【フェーズ A：式目ガードレールの Ruby 化】最優先
  A-1. 句去の間隔チェック（部立リスト＋数値ルール）
  A-2. 句数チェック（季の連続上限・春秋の最低3句）
  A-3. 一座一句物の重複チェック（禁止語リスト）
  A-4. 花・灯火の例外ルール

【フェーズ B：辞書整備】高優先
  B-1. 季語マスタ（既存の和歌DBふりがなを活用）
  B-2. 部立辞書（★体用フラグ付き ― 仕様1）

【フェーズ C：LLM プロンプトの高度化】中優先
  C-1. 感覚モダリティ・視点ヒントの注入
  C-2. 序破急ステージのヒント注入
  C-3. 恋・述懐の文脈管理

【フェーズ D：セッション管理】将来
  D-1. 折・段階の状態保持
  D-2. 折跨ぎの文脈（恋の波・季の負債）
  D-3. kaishi 形式の記録出力
```

## 3-4. 「枯れてから足す」順序での着手案

既存の `KuValidator`（A層のモーラ判定）が動いている。次に足すべきは：

**ステップ1：** A-1（句去チェック）を `KuValidator` とは別クラス `ShikimokuChecker`（仮）として新規作成。部立リストを引数に取り、直近N句との間隔を数値で判定する純粋な Ruby ロジック。LLM 不要なので確実に「枯れる」。

**ステップ2：** B-2（部立辞書）を体用フラグ付きで整備。水無瀬三吟の全100句が「正解データ」として使える。各句の部立を辞書で判定させ、手分析の結果と一致するか検証する。

**ステップ3：** ステップ1とステップ2が安定したら、RengaChecker（LLM）から式目判定の負荷を完全に外し、LLM は季語認定の補助と付合生成（C層）に専念させる。

# おわりに ― この分析が示したこと

水無瀬三吟百韻の全100句分析を通じて、三つのことが確認できた。

1. **百韻は一つの設計された世界である。** 季節の弧・序破急・恋と述懐の配置・発句と揚句の円環。宗祇は詳細な楽譜ではなく「骨法」を持ち、その中で即興した。当麻曼荼羅のように、部分と全体が相互に規定し合う構造。

2. **付合の「面白さ」は100%がC層（LLM領域）にある。** Ruby は式目違反を防ぐガードレールであり、付合を生成する力は持たない。だが、そのガードレールがあるからこそ、LLM は安心して詩的飛躍に集中できる。

3. **手分析の結果がそのまま実装の正解データになる。** 全100句の部立・季・付合型の分析は、辞書整備の検証データであり、プロンプト設計の手本であり、生成結果の評価基準である。

この三つの知見をもって、waka-collector の連歌生成は「式目を守りながら面白い」という連歌本来の理想に近づける。
