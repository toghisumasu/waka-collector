## 完了事項

### D-22-1　RengaGenerator の ±1音許容設計を意図的設計として記録

- `(mora - target_mora).abs <= 1` は LLM 生成安定性のための意図的設計
- `mora == target_mora` への厳格化は生成失敗率上昇・フィードバックループ機能不全リスクあり → 変更しない
- `KuValidator#validate` の `warning` はユーザー向け参考情報であり、自動生成ループの採用判定とは責務が異なる設計分離
- 変更条件：字余り句が連歌品質に実害を与えると観測されてから（枯れてから足す）

### D-22-2　ひらがな表記の神祇語は bui: nil として通過する

- `やしろ`（かものやしろの）等はひらがな表記のため `detect_bui` が nil を返す
- 現在「神祇」は `forbidden_bui` に指定ケースなし → 実害なし・対処しない
- 将来「神祇」を `forbidden_bui` に追加する際は `bui_dictionary.yml` へのひらがな表記語追記が必要
