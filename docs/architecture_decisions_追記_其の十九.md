### D-19-1　「短歌一首になるような続き」プロンプトの設計意図

**判断：変更禁止。意図的な設計である。**

`build_full_prompt` の冒頭にある「前の句と合わせて短歌一首になるような続きを作れ」という指示は、連歌の式目に反する表現に見えるが、これは誤りではない。

**設計意図：**  
`前句（17音）＋付句（14音）＝短歌31音`、あるいは  
`前句（14音）＋付句（17音）＝短歌31音`

LLMが「短歌として完結させる」という枠組みで考えることで、文語の自然な接続と情景の完結が担保される。これを「連歌の付け句を作れ」に変えると、接続の自然さが損なわれることが実験で判明している。

**結論：** 将来この行に触れる際は必ず本記録を確認すること。変更は禁止。

### D-19-2　KIGO_BUI / BUI_EXAMPLE_WORDS 定数の責務分離

**判断：季語フィルタリングと禁止語説明を定数で分離する。**

| 定数 | 責務 | 使用箇所 |
|:--|:--|:--|
| `KIGO_BUI` | 季語→部立の対応（B層フィルタ用） | `kigo_hint` メソッド内で `forbidden_bui` と突合 |
| `BUI_EXAMPLE_WORDS` | 部立→具体語リスト（C層プロンプト用） | `build_full_prompt` 内で禁止語をLLMに伝える際に展開 |

従来は `forbidden_bui` に部立名（「降物」など）をそのまま渡していたが、
LLMが「降物」という概念語を理解できない場合があった。
`BUI_EXAMPLE_WORDS` で「雨・雪・露・霜・時雨」などの具体語に展開することで
禁止語の精度が向上した。

### D-19-3　kigo_hint の設計方針

**判断：season_hint × forbidden_bui の積集合で季語候補を決定する。**

```ruby
# 設計の骨子
SEASON_WORDS[season_key]
  .reject { |w| KIGO_BUI[w] && forbidden_bui.include?(KIGO_BUI[w]) }
  .reject { |w| @maeku.include?(w) }
  .shuffle.first(2)
```

- `SEASON_WORDS` から現季節の語を取得
- `KIGO_BUI` で部立を引いて `forbidden_bui` に含まれる語を除外
- 前句に既に出現している語を除外
- ランダムに2語を選びプロンプトに注入

**注意：** 雑句（season_label == "雑"）の場合は季語注入を行わず、
「#{season_label}の情趣を詠むこと」もスキップする。

### D-19-4　Pythonパッチスクリプト方式の廃止

**判断：其の二十以降は Claude Code を使用する。Pythonパッチは使わない。**

其の十九において、Pythonによるファイル編集スクリプトは以下の問題を引き起こした：

1. `\#{}` のエスケープ問題（PythonとRubyの埋め込み式の競合）
2. インデント（2スペース vs 4スペース）の思い込みミス
3. 定数追加による行番号ずれへの対処漏れ
4. ヒアドキュメントとシェル展開の衝突

本来5分の作業が1時間超に及んだ。

**代替：** Claude Code（`npm install -g @anthropic-ai/claude-code`）を Mac mini 上で
直接実行する。ファイルを直接読み・`str_replace` で正確に編集するため
上記問題が根絶できる。

**移行後も維持するルール：**
- 日本語を含むファイルの編集は SCP 転送 or Claude Code を使用
- `cat > file << 'EOF'` でのヒアドキュメント方式は日本語ファイルのみ継続許可
- `*.py` パッチスクリプトは `.gitignore` 対象、リポジトリには含めない

### D-19-5　ng「句数不足（春・秋3句以上必要）」の誤判定について

**判断：現時点では対処しない。RengaChecker の設計上の既知問題として記録する。**

**観測：** 実動作で「句数不足（春・秋：3句以上必要）」の ng が頻発する。

**原因：** `RengaChecker` が前句と付句の2句だけを見て句数を判定している。
百韻全体での連続句数カウントと混同しており、孤立した秋2句を見て「3句ない」と判定する。

**方針：**
- 生成品質の問題ではない。チェッカーのロジック問題。
- 連鎖全体の文脈（直前N句の季節履歴）を `RengaChecker` に渡す修正が必要だが、
  現時点で実害（誤った句が連歌に採用される）は発生していない。
- 「枯れてから足す」原則に従い、実害が観測されるまで対処を保留する。
- 修正する場合は `ShikimokuChecker#next_constraints` の季情報を
  `RengaChecker` のプロンプトにも渡す形が最小変更。
