# 付合型の役割分担分析
# 水無瀬三吟百韻から帰納する「Rubyで解ける付合」と「LLMが担う付合」

**位置づけ:** `minase_analysis.md` の補論  
**目的:** 付合パターンを「決定論的処理（Ruby/MeCab）」と「創造的判断（LLM）」に分類し、役割分担の設計根拠とする  
**最終更新:** 2026-06-18

---

## 基本方針

```
決定論的に判定できる → Ruby / MeCab が担う
  └ 条件が明示的・客観的・語彙ベースで解ける

創造的判断が必要 → LLM が担う
  └ 文脈・情感・詩的転換の評価が必要
```

COBOLアナロジーで言えば：
- Ruby = 検査プログラム（条件判定・バリデーション）
- LLM = 承認者（判断・生成）
- 両者の間に明確なインタフェース（渡すデータの仕様）を設ける

---

## 分類表

### カテゴリA：Rubyで完全に解ける（LLM不要）

| # | 付合の要素 | 判定方法 | 根拠 |
|:---|:---|:---|:---|
| A-1 | **句去（部立の間隔）** | 直近N句の部立リストと照合 | 式目の数値ルール。降物三句去・植物五句去など |
| A-2 | **同季の連続上限** | 直近の季フラグをカウント | 春秋五句・夏冬三句の上限 |
| A-3 | **季の最低句数** | 季切替時に直前の連続数を検査 | 春秋は「三句に至らざれば用いず」 |
| A-4 | **一座一句物の重複** | 出現語と禁止語リストを照合 | 郭公・砧・女など一座に一度限り |
| A-5 | **打越（二句前との同類衝突）** | 二句前の部立と現句を照合 | 「時分と時分」「降物に聳物」など |
| A-6 | **モーラ数の正否** | MeCabで読み取得→モーラカウント | 長句17・短句14（既にKuValidatorで実装済み） |
| A-7 | **句種の交互** | 直前句の種別を参照 | 長句→短句→長句の交互規則 |
| A-8 | **語の重複（輪廻）** | 直近句の語幹セットと照合 | 「雪に富士を付けて又氷室不可然」など類語 |

**実装状況：**
- A-6：`KuValidator` で実装済み ✅
- A-1〜A-5, A-7, A-8：未実装、今後 `RengaChecker` または新クラスで対応

---

### カテゴリB：Rubyで「候補の絞り込み」、LLMで「最終判断」

| # | 付合の要素 | Rubyの役割 | LLMの役割 |
|:---|:---|:---|:---|
| B-1 | **季語の認定** | 季語辞書との照合（既知語） | 辞書にない語・文脈依存の季節判定 |
| B-2 | **部立の判定** | 体言・用言を抽出して部立辞書と照合 | 「涙の時雨」など分別物の判定 |
| B-3 | **語の重複フィルタ** | 核（主部）の読み語幹を抽出 | 「ゆかり」的な意味的重複の検出 |
| B-4 | **本歌の三句連続** | 本歌フラグが立った句の連続数を検査 | どこからどこまでが本歌の影響圏か |

**設計方針：**
Rubyが「これは季語辞書にある語か」を先に判定し、辞書ヒットなら即確定。
辞書ミスのみLLMに問い合わせる。LLMの呼び出し回数を最小化する。

---

### カテゴリC：LLMが担う（Rubyでは解けない）

| # | 付合の要素 | なぜRubyで解けないか | LLMへの指示の粒度 |
|:---|:---|:---|:---|
| C-1 | **感覚モダリティの転換** | 「この句は視覚か聴覚か」の判定が文脈依存 | 前句の感覚種別を伝え、異なる感覚で生成させる |
| C-2 | **視点の遠近・高低の転換** | 「遠景か近景か」の判定が詩的解釈を要する | 前句の空間的性質を伝え、反対方向に転じさせる |
| C-3 | **情感の転換（景→事・事→景）** | 「景物句か心情句か」の判定が意味解釈を要する | 前句の体/用を判定し、反転を促す |
| C-4 | **付合の「詩的飛躍」** | 7→8句の「草枯れ→道が見える」のような写実的連想 | 前句のキーワードから物理的・論理的帰結を生成 |
| C-5 | **無常・述懐の深度調整** | 述懐の「重さ」が文脈に適切かの評価 | 直前の述懐句数を渡し、深めるか軽くするか指示 |
| C-6 | **雅致・品位の維持** | 連歌の格調が保たれているかの評価 | プロンプトの基本トーン設定で対応 |

## 判定フローの設計案

```
前句受け取り
    │
    ▼
【Ruby層①：形式検査】
  - モーラ数（KuValidator）
  - 句種（長句/短句）の交互確認
    │
    ├─ NG → 即エラー返却（LLM不要）
    │
    ▼
【Ruby層②：式目検査】
  - 部立を辞書照合で分類
  - 句去リストと照合（A-1〜A-5）
  - 一座一句物チェック（A-4）
    │
    ├─ NG → 禁止部立リストをLLMに渡して再生成依頼
    │
    ▼
【Ruby層③：候補生成前処理】（B層）
  - 季語の辞書照合
  - 使用済み語幹のセット化
  - 許可部立・禁止部立のリスト化
    │
    ▼
【LLM層：生成】（C層）
  インプット：
    - 前句本文
    - 求める句種（長/短）
    - 求める季（または雑）
    - 禁止部立リスト
    - 禁止語幹リスト
    - 前句の感覚種別ヒント（視覚/聴覚/嗅覚/心情）
    - 前句の空間ヒント（遠/近/高/低）
  アウトプット：
    - 候補句本文
    - 使用季語
    - 部立
    │
    ▼
【Ruby層④：事後検査】
  - 生成句のモーラ数検査
  - 生成句の部立が禁止リストに抵触しないか再確認
    │
    ├─ NG → 再生成依頼（最大3回）
    │
    ▼
表示・確定
```

## 水無瀬三吟の実例による検証

### 初折表8句での各カテゴリ出現

| 転じ | A層で解決 | B層で絞込 | C層が担当 |
|:---|:---|:---|:---|
| 1→2 | 春継続OK、降物→水辺（句去クリア） | 梅=春の確認 | 感覚を視覚→嗅覚に転換（C-1） |
| 2→3 | 春継続OK、水辺→植物（句去クリア） | 柳=春の確認 | 空間を里→川辺に絞り込む（C-2） |
| 3→4 | 春→雑の転季、植物→水辺（句去クリア） | — | 感覚を視覚→聴覚に転換、時間飛躍（C-1） |
| 4→5 | 雑→秋の転季OK、時分の三句去クリア | 月=秋・霧=秋の確認 | 春から秋への大転換を詩的に着地（C-3） |
| 5→6 | 秋継続OK、光物→降物（句去クリア） | 霜=冬境の確認 | 視点を空→地に降下（C-2） |
| 6→7 | 秋継続OK、降物→動植物（句去クリア） | 虫=秋の確認 | 無機景に生命を介入（C-4） |
| 7→8 | 秋→雑の転季、動物→居所（句去クリア） | — | 草枯れの物理的帰結として道が見える（C-4） |

**観察：** A層（式目検査）は全転じで必ず発動する。C層は「感覚転換」と「視点移動」が最頻出。

## 今後の実装優先順位

```
優先度 高
  └ A-1（部立句去）：RengaCheckerのRuby化
  └ A-2/A-3（季の句数）：RengaCheckerのRuby化
  └ B-1（季語辞書）：季語マスタテーブルの整備

優先度 中
  └ B-3（語幹重複フィルタ）：既に課題として認識済み（ゆかり問題）
  └ A-4（一座一句物）：禁止語リストをRuby定数で管理
  └ C-1（感覚モダリティ）：LLMプロンプトへのヒント注入

優先度 低（将来フェーズ）
  └ C-2（視点遠近）：プロンプトの洗練
  └ C-5（述懐深度）：文脈管理の高度化
  └ C-6（雅致）：評価モデルの検討
```

## 未解決の問題（要継続分析）

1. **「分別物」の扱い** — 「花の雪」は植物に嫌い降物に嫌わず、など複合部立の処理
2. **体用の判定** — 「岡・嶺は山の体、瀧・杣は山の用」をどう実装するか
3. **感覚種別の自動判定** — MeCabで動詞・感覚語を抽出して視覚/聴覚を判定できるか
4. **恋句の連続管理** — 恋は五句まで連続可だが、恋の「深度」は変化させる必要がある
