唯一の失敗ケース（verse71→72）は式目チェッカーが「句数:秋」（kukazo_under、季switch時の最短規制未達＝D-19-5の既知問題）で5回連続ng判定し、forced_zatsuへ救済された。forced_zatsuで生成された句（「月の光りゆかし夜の夢にふける」）がたまたま秋の季語を含んでいたため季自体は転換できず、count=6まで伸びた上で直後のverse73でようやく雑へ転換した。filter_pool自体の不具合ではなく、「switch先を弾かれた末に非季寄りの語を生成し続けた」LLM側の挙動起因である可能性が高い。

## 2-4. 夏句の出現状況

98句（全99句）中、夏の季語を含む句は**0件**。

### compute_season_hintの実装確認

- `app/data/kukazo_rules.yml`：夏は`max: 3`のみ定義（`min`キーなし）。仕様上は夏が候補に入ることを妨げていない。
- `RengaGenerator#filter_pool`（`app/services/renga_generator.rb:334`）：`must_switch`時は`pool.reject { |s| s[:season] == hint[:current] }`＝現在季（秋）を除くだけで、夏を含む他季は等しく候補に残る。夏を排除するロジックは存在しない。

### 実際の種プール（Waka DB）の季分布を検証

`build_seed_pool`が参照する`Waka`テーブル（upper/lower_phrase_textの両方が非空、2,527件）を`compute_season_tag`と同一ロジックで再集計：

| 季 | 件数 | 割合 |
|---|---|---|
| 雑 | 1,480 | 58.6% |
| 秋 | 504 | 19.9% |
| 冬 | 183 | 7.2% |
| 春 | 243 | 9.6% |
| **夏** | **117** | **4.6%** |

夏は候補として存在するが、母集団の4.6%と最も少ない。さらにmust_switch発火時の候補プールは「現在季（秋）以外全部」（雑・春・夏・冬の合計2,023件相当）であり、この中で夏が占める比率は約5.8%（117/2,023）。本runで季転換が成功したのは6回のみ（2-3参照）であり、期待値计算では6回中の夏出現期待値は約0.35件——**0件という結果は統計的に十分あり得る範囲**（6回とも外れる確率は約(1-0.058)^6≈70%）。

**結論：** 夏が出現しない主因はcompute_season_hint/filter_poolの排他ロジックではなく、(1) 季転換の成功回数自体が6回とサンプルが小さいこと、(2) 母集団中の夏候補の比率が他季より低いこと、(3) filter_poolのmust_switch分岐が「現在季を除外するだけ」で転換先を積極選択しない設計であるため、多数派の雑に転換先が偏りやすいこと、の複合と考えられる。「未出現＝バグ」ではなく「出にくい設計＋小サンプル」が主因であり、これ自体を修正する優先度は低いと判断する（実害未観測・[[production_code_frozen]]の「枯れてから足す」原則に照らして温存が妥当）。
